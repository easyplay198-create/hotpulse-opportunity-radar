# HotPulse 当前研究沉淀

## 1. 核心研究结论
HotPulse 的核心风险不是数据源不够多，而是容易做成大而全数据聚合平台。

项目应避免追求“更多榜单、更多数据、更多图表”的表层完整性，而应聚焦：哪些情报和信号能直接支持海外市场进入前的“验证 / 观察 / 暂不进入”判断。

## 2. 使用频率陷阱
产品最大风险之一是使用频率陷阱。

如果只做 MVP 前验证工具，用户只会在准备立项、评估市场或准备出海时打开，使用频率天然偏低。低频工具即使单次价值高，也容易在早期缺少持续反馈、持续使用和持续转化。

因此，HotPulse 需要“出海企业情报站 / 今日出海机会简报”作为外层入口，用每日 / 每周市场变化、机会变化、风险变化、案例变化，提高复访频率。

## 3. 双层产品结构
### 3.1 外层：出海企业情报站
负责让用户每天 / 每周愿意回来，快速知道：
- 今天有什么新机会
- 哪些方向正在升温
- 哪些机会值得验证
- 哪些风险正在上升
- 有哪些案例或商业模式值得借鉴

### 3.2 内层：MVP 前验证系统
负责承接真正的决策与付费转化，帮助用户形成：
- 可追溯机会假设
- 个性化进入匹配
- 7 天 MVP 验证方案
- 执行服务需求

## 4. 替代风险
ChatGPT / Perplexity 是 HotPulse 最大替代风险。

如果 HotPulse 只是泛资讯、泛趋势总结、泛市场调研入口，用户完全可以通过通用大模型获得近似结果。HotPulse 不能和大模型比信息广度，而应在专项判断与可执行链路上形成差异化。

## 5. 差异化来源
HotPulse 的差异化来自三点：

1. 证据链
   - 机会必须有 evidenceChain。
   - 证据类型、来源、强度和局限性要明确。
   - 没有真实 evidence 的机会不能生成。

2. 出海专项维度
   - 支付适配
   - 本地化成本
   - 上架合规
   - 获客投流
   - AI token 成本结构
   - 竞争压力
   - 证据强度
   - 进入适配度

3. 执行资源
   - 海外支付与订阅支付
   - 本地小额代付
   - 小语种翻译
   - 低价 token
   - Google Play / App Store 合规上架与过包
   - 投流 / 广告获客验证资源

这些资源不应写成销售广告，而应产品化为诊断维度与执行服务入口。

## 6. 机会转译原则
资讯、案例、趋势和市场信号本身不是机会。HotPulse 需要把它们转译为机会假设，并回答：
- 这条信号说明什么
- 和哪个市场有关
- 和哪类企业有关
- 潜在机会是什么
- 风险是什么
- 可以怎么验证
- 证据来源是什么

## 7. 当前研究判断
- 作为免费情报入口：有价值。
- 作为单独 SaaS 订阅：早期粘性和付费不够稳。
- 作为 MVP 前验证报告入口：有商业机会。
- 作为支付 / 本地化 / 投流 / 上架服务入口：更有变现空间。

因此，当前策略应是：出海企业情报站提高打开频率，MVP 前验证报告承接付费转化，执行服务承接后续变现。

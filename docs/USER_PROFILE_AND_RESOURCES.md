# HotPulse 用户画像与资源维度

## 1. 总目标用户
所有准备出海、正在评估海外市场、需要做海外进入前验证或海外增长决策的企业 / 团队。

## 2. 早期优先用户
- 3-10 人 AI 应用 / 工具 / SaaS 出海团队
- 已有产品、想测试海外市场的公司
- 游戏 / 短剧 / 语聊内容产品团队

“小团队”只是早期优先用户之一，不是产品总定位。

## 3. 用户现有优势
- HaiPay 海外支付
- 全球本地支付与订阅支付
- 本地小额代付
- 小语种翻译
- 低价 token
- Google Play / App Store 合规上架与过包
- 投流 / 广告获客验证资源

## 4. 产品化诊断维度
这些资源不要写成销售广告，而要产品化为诊断维度：
- 支付适配
- 本地化成本
- 上架合规
- 获客投流
- AI 成本结构
- 竞争压力
- 证据强度
- 进入适配度

## 5. 顾问匹配使用方式
机会顾问应结合企业自身条件判断：
- 当前产品阶段是否适合进入
- 预算是否支撑最小验证
- 支付与订阅能力是否匹配目标市场
- 本地化成本是否可控
- 上架和合规风险是否可处理
- 获客投流资源是否足以完成验证
- token 成本是否影响商业模型
- 竞争压力是否过高
- 证据强度是否足以支持行动

## 6. 表达原则
- 不写成销售广告。
- 不夸大资源能力。
- 不把服务资源直接包装成机会结论。
- 资源应服务诊断、匹配、验证和执行承接。

# HotPulse 决策日志

## Day 23.1 关键决策
- 不要把项目拉回课程原版“三平台热榜”。
- 不要再做 `/api/hot/weibo`。
- 课程里的微博真实热搜任务，在本项目中等价替换为 `/api/opportunities?source=real`。
- 热门产品不是机会本体，热门产品只是证据、标杆竞品或市场信号。
- 大厂产品不能直接作为机会标题，只能作为证据、标杆竞品或市场信号。
- 机会标题必须是机会假设，不能直接用大厂产品名。
- 没有真实 evidence 的机会不能生成。
- 没有直接评论证据时，必须写“推断痛点 / 验证假设 / 置信度”。
- 首页第一屏后续优先是今日出海机会简报。
- 首页不要堆完整报告。
- 报告页负责解释。
- 复制文本负责交付。
- 顾问负责个性化匹配。
- 当前商业策略：先卖验证结果，不先卖订阅。

## 定位纠偏
HotPulse 当前定位统一为：出海企业情报站 + MVP 前市场验证与机会决策系统。

不是旧版热点速判工具，不是内容创作者选题工具，不是普通 AI 热榜，不是泛资讯站，也不是只针对小团队。

## API 决策
- 当前主接口是 `/api/opportunities`。
- 当前真实来源统一是 `/api/opportunities?source=real`。
- `/api/hotspots` 不再作为当前主接口。
- `/api/hot/weibo` 不再进入本项目路线。

## 首页决策
首页后续应优先服务：
1. 今日出海机会简报
2. 今日可追溯机会发现
3. 案例与模式观察
4. 我的机会顾问
5. 市场信号榜

市场信号榜是原始信号池，不等同于可直接进入机会。

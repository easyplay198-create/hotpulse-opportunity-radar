# HotPulse 产品原则

## 1. 定位原则
- HotPulse 不是泛资讯站，而是机会化情报站。
- 不做大而全数据聚合平台。
- 不做通用市场验证工具。
- 不和 Sensor Tower 比数据深度。
- 不和大模型比信息广度。

## 2. 情报转译原则
每条资讯都要尽量转译成：
- 机会解读
- 适合谁看
- 可验证方向
- 风险提示

不只展示哪里热，要展示哪里有可验证切口。

## 3. 首页原则
- 首页第一屏优先展示“今日出海机会简报”。
- 情报层负责提高复访，验证报告负责变现。
- 首页不要堆完整报告。
- 报告页负责解释，复制文本负责交付，顾问负责个性化匹配。

## 4. 机会与证据原则
- 热门产品不是机会本体，热门产品是证据。
- 机会必须可追溯。
- 没有真实 evidence 的机会不能生成。
- 推断必须标明是推断。
- 市场信号榜是原始信号池，不等同于可直接进入机会。
- 没有直接评论证据时，必须标记为推断痛点或验证假设。

## 5. 决策原则
每个机会都应尽量支持用户判断：
- 是否值得验证
- 是否只适合观察
- 是否暂不进入
- 适合什么类型的企业
- 风险在哪里
- 可以如何低成本验证

## 6. 边界原则
- 不把项目拉回旧版“三平台热榜”。
- 不把大厂产品名直接写成机会标题。
- 不把单条弱信号当作结论。
- 不用泛泛总结替代 evidenceChain。

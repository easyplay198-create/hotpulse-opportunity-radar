# PRAXON（派克森商机雷达）PRD

## 1. 核心定位
PRAXON（中文名：派克森，传播名：派克森商机雷达）是“AI驱动的出海商机发现与验证系统”。

外层是出海企业情报站，用“今日出海机会简报”、案例观察、风险提示和市场信号更新提高打开频率。内层是 MVP 前验证系统，用可追溯机会发现、个性化机会顾问和验证报告承接付费与服务转化。

机会雷达不是终点，而是把资讯、案例、趋势和市场信号转译成机会假设。验证报告输出可执行的 7 天 MVP 验证方案。执行服务入口承接支付、本地化、token 成本、上架、投流等服务。

## 2. 产品闭环
出海情报 → 信号识别 → 机会发现 → 个性化匹配 → 验证报告 → 执行服务。

### 2.1 出海情报
通过每日 / 每周情报、案例与模式观察、风险变化提示，让用户形成持续打开习惯。

### 2.2 信号识别
从产品信号、社区信号、开发者信号、趋势信号、案例信号和资源信号中识别值得观察的市场变化。

### 2.3 机会发现
将资讯、案例、趋势和市场信号转译为机会假设，并通过 evidenceChain 与 evidenceStrength 标注证据来源和置信度。

### 2.4 个性化匹配
结合用户的产品阶段、预算、团队能力、支付能力、本地化能力、token 成本、上架能力和投流资源，判断机会是否适合进入。

### 2.5 验证报告
输出可执行的 7 天 MVP 验证方案，帮助用户明确验证目标、渠道、素材、预算、指标、风险和退出条件。

### 2.6 执行服务
在验证方案之后，承接支付、本地化、token 成本、Google Play / App Store 上架、投流获客等服务需求。

## 3. 目标用户
所有准备出海、正在评估海外市场、需要做海外进入前验证或海外增长决策的企业 / 团队。

### 3.1 早期优先用户
- 3-10 人 AI 应用 / 工具 / SaaS 出海团队
- 已有产品想测试海外市场的公司
- 游戏 / 短剧 / 语聊内容产品团队

“小团队”只是早期优先用户之一，不是 PRAXON 的总定位。

## 4. 当前首页方向
1. 今日出海机会简报
2. 今日可追溯机会发现
3. 案例与模式观察
4. 我的机会顾问
5. 市场信号榜

首页第一屏后续优先服务“今日出海机会简报”，解决用户复访与使用频率问题。市场信号榜是原始信号池，不等同于可直接进入机会。

## 5. 核心价值
PRAXON 帮助用户回答：
- 今天有哪些海外市场变化值得知道？
- 哪些变化可能形成可验证机会？
- 哪些机会适合我的产品、预算和资源？
- 应该如何在 7 天内完成 MVP 前验证？
- 验证后需要哪些执行资源支持？

## 6. 当前不是
PRAXON 当前不是：
- 不是旧版热点速判工具
- 不是内容创作者选题工具
- 不是大而全数据聚合平台
- 不是 Sensor Tower 替代品
- 不是通用市场调研工具
- 不是纯服务商中介页
- 不是普通 AI 热榜
- 不是泛资讯站
- 不是只针对小团队

## 7. MVP 阶段原则
- 先验证出海机会情报与 MVP 前验证链路是否成立。
- 不追求大而全数据覆盖。
- 不把热门产品直接当成机会本体。
- 机会必须可追溯，有 evidenceChain 支撑。
- 没有真实 evidence 的机会不能生成。
- 推断必须标注为推断痛点 / 验证假设 / 置信度。
- 每个机会都应尽量对应“适合谁看、为什么值得观察、如何验证、主要风险是什么”。

## 8. 商业化方向
- 免费情报入口：降低使用门槛，提高打开频率。
- 案例和用户访谈：建立信任与场景理解。
- ¥699-¥1499 单份验证报告：承接明确机会判断需求。
- ¥2999-¥4999 MVP 验证包：承接更完整的验证执行。
- 支付、本地化、token 成本、上架、投流等执行服务：承接后续变现。

当前商业策略是先卖验证结果，不先卖纯 SaaS 订阅。

## 9. 成功标准
- 用户能快速理解 PRAXON 是“AI驱动的出海商机发现与验证系统”。
- 用户愿意因为市场变化、机会变化、风险变化和案例变化持续打开。
- 用户能从信号看到机会假设，并理解证据来源。
- 用户能基于顾问匹配和验证报告决定“验证 / 观察 / 暂不进入”。
- 验证报告能形成可交付价值，并自然连接执行服务。

## 10. 关键风险
- 容易被拉回旧版“三平台热榜”或“热点速判工具”。
- 容易做成大而全数据聚合平台。
- 只做 MVP 前验证工具会低频，存在使用频率陷阱。
- ChatGPT / Perplexity 等通用大模型会替代泛信息检索。
- 如果没有证据链、出海专项维度和执行资源，差异化不足。
